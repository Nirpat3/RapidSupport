# Troubleshooting for PAX Terminal

**[Troubleshooting for PAX Terminal]{.underline}**

**1) [Identify the IP Address]{.underline} :-**

Ensure the IP addresses of both the Pax terminal and the iPad are within the same network family (e.g., xxx.xxx.xxx.109). All the \"x\"s should match on both the iPad and the Pax terminal.

**2)** **[Verify ECR Settings in the Pax Application]{.underline} :-**

**\*\*For Pax S300:\*\***\
** -** Go to ECR Settings.\
 - Verify the following settings:\
 - Connection Type: Ethernet\
 - Port: 10009\
 - Protocol Type: HTTP Get or HTTPS Get\
\
🡪Ensure the protocol type matches the settings on the iPad under Pax Settings.\
\
🡪If, after verifying these steps, the issue persists, restart the iPad, obtain the error code, and contact the payment support manager.

**\*\*For Pax A35:\*\*\**
- Go to Network Settings and ensure that only Wi-Fi or Ethernet is toggled on. If both are toggled on, it will cause issues.

\- Verify ECR Settings:

\- Access the menu by tapping the four corners of the screen in the following order: top left, top right, bottom right, bottom left.

\- Go to Host Settings.

\- Ensure Ethernet is toggled on.

\- Set Protocol Type to HTTP Get or HTTPS Get.\
\
🡪Ensure the protocol type matches the settings on the iPad under Pax Settings.\
\
🡪If, after verifying these steps, the issue persists, restart the iPad, obtain the error code, and contact the payment support manager.
