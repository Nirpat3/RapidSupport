# How load and redeem calculated in report_

**How load and redeem calculated in report?**

In report, there is one section available for gift card (Type of payment) terms and calculation.

![](media/image1.png){width="6.5in" height="4.75in"}

When you add amount in new or existing gift card, it sales will be appear in load section from Shift report.

![](media/image2.png){width="6.5in" height="4.819444444444445in"}

Gift card payment can be done by other tender type excepting gift card tender type.

![](media/image3.png){width="6.5in" height="4.916666666666667in"}

When you complete any transaction with gift card, it amount will appear as redeem in the shift report.

![](media/image4.png){width="6.5in" height="4.722222222222222in"}

![](media/image5.png){width="6.5in" height="4.791666666666667in"}

GC liability is total Gift card amount that is in balance after load / redeem. In short GC liability is gift card balance.

In load and redeem, there is one filed available titled as count it represents number of count that how many times load and redeem process done in particular report scope.
